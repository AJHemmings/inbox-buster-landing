# README Design — Inbox Buster Landing Page Repo
**Date:** 2026-03-04

---

## Goal

Replace the default Next.js README with a professional product advertisement for Inbox Buster.
Dual purpose: product showcase (primary) + portfolio piece (secondary).

## Approach

Option A — Product-first, portfolio footnote. The README reads like a product page.
Dev context lives at the bottom as a brief footnote.

## Structure

1. **Banner** — placeholder image slot (`public/readme-banner.png`)
2. **Wordmark + tagline** — ⚡ INBOX BUSTER / "Your inbox is a disaster. Let's fix that."
3. **Platform badges** — shields.io with brand colors (#8B5CF6 purple, #4ADE80 green)
4. **See It In Action** — two side-by-side screenshot placeholder slots
5. **Features** — 6-feature table with emoji icons
6. **Pricing** — three-tier table with Stripe Climate note
7. **Footer CTA** — "Your inbox won't clean itself." + waitlist link
8. **Dev footnote** — tech stack badges (Next.js, Tailwind, TypeScript, Lucide, Vercel)

## Brand Constraints

- GitHub strips inline CSS — colour comes from shields.io badges and tone/copy
- Lightning bolt ⚡ used as brand accent throughout
- Copy matches landing page: bold, punchy, no corporate fluff
- Prices and waitlist URL are placeholders until launch
